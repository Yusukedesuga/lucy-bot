# Lucy Deps: Discord AIアシスタント Bot
### オンラインコミュニティの課題解決を目的とした、LLM連携 Botの開発と運用

---

## 1. プロジェクト概要

このプロジェクトは、多人数が参加するゲームコミュニティにおける**「パーティ募集の非効率性」**と**「攻略情報の検索コスト」**という2つの課題を解決するために開発されました。

Gemini API（LLM）の高度な自然言語処理能力を活用し、ユーザーの曖昧な会話から意図を正確に読み取り、最適なアクション（募集パネルの生成、攻略情報の検索・要約など）を即座に実行する Bot です。

## 2. 実装された主要機能 (Technical Highlights)

| 機能 | 技術的アピールポイント |
| :--- | :--- |
| **対話型パーティ募集** | 自然言語解析（LLM）により、日時やコンテンツ名、ロールを会話から自動抽出。Discordの**インタラクション機能（ボタン）**を利用した**ステートフルなUI**で、参加状況をリアルタイムに管理。 |
| **高度な情報検索・要約** | **DuckDuckGo Search**を連携し、外部サイト（エオルゼアデータベース）を検索。検索結果をLLMに読ませる**RAG（検索拡張生成）の簡易実装**により、情報鮮度と応答精度を両立。 |
| **マクロ・カンペ機能** | ユーザーが登録した複雑なテキスト情報をBotの**内部知識**として管理。対話を通じて必要な部分のみを抽出し、**即時カンペ生成**を実現。 |
| **24時間クラウド運用** | **Render**を利用した**CI/CDパイプライン**を構築。GitHubへのプッシュと同時に本番環境が更新される、24時間稼働可能な**クラウドインフラ**を運用。 |

## 3. システムアーキテクチャ

本プロジェクトは、高い拡張性と保守性を確保するため、機能をモジュールごとに分離した**Cogs（コグ）構造**を採用しています。

- **`main.py`**: Botの起動と各種Cogのロード管理。
- **`cogs/chat.py`**: LLMとの会話インターフェース、プロンプト管理。
- **`cogs/partyfinder.py`**: パーティ募集パネルの生成とボタンイベント処理。
- **`cogs/search.py`**: 外部検索エンジンの実行、結果の取得とLLMへの受け渡し。
- **`utils.py` / `prompts.py`**: 共通処理（時刻計算、マクロ管理）とAIへの指示文を一元管理。

## 4. 開発環境

- **言語:** Python 3.10+
- **フレームワーク:** discord.py
- **LLM:** Google Gemini 2.5 Flash API
- **運用環境:** Render (PaaS)
